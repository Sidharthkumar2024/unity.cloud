Setup Guide

Get unityai.cloud running inside your IDE, coding tool, or AI client with the real local proxy URL, UnityAI model IDs, and compatible endpoints.

Drop-in compatible Per-key budgets Proxy auth UnityAI models

Prerequisites

Node.js 18 or newer

Required for running the setup wizard with npx.

unityai.cloud API key

Create or receive a customer API key before connecting a coding tool.

Supported coding tool

Use Codex, Claude Code, Cline, Kilo Code, OpenCode, VS Code, or a manual client.

Quick Setup

Run the UnityAI setup wizard, paste your API key, then choose the coding tool you want to configure.

Main Command
npx -y @unityai/cloud-cli@latest
Connection Values
API origin: https://unityai.cloud
Anthropic-compatible URL: https://unityai.cloud/v1
OpenAI-compatible URL: https://unityai.cloud/api/v1
API Key: YOUR_API_KEY
Primary model: unityai-cloud
Fast model: unityai-cloud

CLI Tools

Codex CLI

Use the OpenAI-compatible endpoint and set the model to unityai-cloud.

OPENAI_API_KEY=YOUR_API_KEY
OPENAI_BASE_URL=https://unityai.cloud/api/v1
MODEL=unityai-cloud

Claude Code

Use the Anthropic-compatible endpoint with the same API key.

ANTHROPIC_AUTH_TOKEN=YOUR_API_KEY
ANTHROPIC_BASE_URL=https://unityai.cloud/v1
ANTHROPIC_MODEL=unityai-cloud

Cline or Roo Code

Choose Anthropic-compatible mode and point the client to the UnityAI base URL.

{
  "apiProvider": "anthropic",
  "anthropicBaseUrl": "https://unityai.cloud/v1",
  "apiKey": "YOUR_API_KEY"
}

API Reference

Proxy endpoints accept API key authentication through either an API-key header or a bearer token.

POST /v1/messages

Create an Anthropic-compatible message request. Set stream: true for SSE streaming.

GET /v1/models

List the public model IDs available to your account.

POST /api/v1/chat/completions

Create an OpenAI-compatible chat completion with normalized output.

Request Example
{
  "model": "unityai-cloud",
  "max_tokens": 1024,
  "messages": [
    { "role": "user", "content": "Hello, UnityAI" }
  ],
  "stream": false
}

Available Models

unityai-cloudAuto routing model for balanced production output.
unityai-fastFast model path for lightweight tasks and UI helpers.
unityai-reasoningDeep analysis path for planning, code review, and decisions.
unityai-visionImage and document understanding path for visual workflows.

Troubleshooting

Client returns unauthorized

Check that the API key is present and that the correct compatible endpoint is configured.

Model is not found

Use /v1/models or /api/v1/models to confirm the IDs enabled for the account.

Streaming does not display

Confirm that your client supports server-sent events and that stream is set to true.
